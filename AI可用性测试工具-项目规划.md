# AI 可用性测试工具（EyeUX / 暂定名）

> 用户说好 ≠ 真的好。眼睛不会撒谎。
> WebCam 眼动追踪 + AI 分析，让可用性测试人人可用。

---

## 🎯 核心理念

用 WebCam 眼动追踪替代昂贵的眼动仪硬件，让任何人都能参与可用性测试。AI 做 UX 研究员的工作——不只给数据，而是告诉你**哪里有问题、为什么有问题、怎么改**。

---

## 🏗️ 技术架构（三层）

### 采集层（前端）
- **WebGazer.js** — WebCam 眼动追踪（校准后可达到100-200px精度）
- **MediaPipe Face Mesh** — 面部468个关键点，眼睛区域提取
- **Tobii Pro SDK** — 高精度模式（专业版，1-2px精度）
- **鼠标追踪** — 兜底方案（无摄像头时）
- **浏览器扩展** — 测试任意网页（可选）

### AI 分析引擎（后端）
- 注视点聚类（DBSCAN / GMM）
- 扫视/注视分类（I-VT / I-DT 算法）
- **困惑/迷失检测**（序列模型/LSTM — 反复扫视 + 长注视 + 鼠标徘徊）
- 视觉显著性预测
- 个人眼动基线建模
- **LLM 分析报告生成**（把数据转成人话 + 优化建议）

### 展示层（前端）
- React 热力图叠加
- 用户眼动轨迹回放
- AOI（兴趣区域）分析
- Figma 插件集成
- 自动生成 PDF 报告

## 🧠 ML 核心场景

### 1. 困惑检测
```
用户在同一个区域反复扫视 3 次以上
+ 注视时长远超基线
+ 鼠标在附近徘徊但不点击
→ AI判断：用户在这里卡住了
→ 自动标注 → 给出优化建议（如"CTA按钮文案可能不清晰"）
```

### 2. 视觉层级评估
```
10个用户测试你的落地页
→ Hero区域被 100% 用户看到 ✅
→ CTA按钮被 70% 用户看到 ⚠️
→ 第三屏几乎没有用户滚动到 ❌
→ 关键卖点在 3秒内平均只被注视 0.2秒 ❌
→ 自动建议：把CTA上移、简化首屏、卖点前置
```

### 3. 意图 vs 实际行为
```
用户说："我觉得这个页面很清楚"
但眼动数据显示：他在导航栏来回扫了8秒才找到入口
→ AI结论：用户以为自己理解了，但行为暴露了真实困惑
→ 这是问卷调查永远发现不了的问题
```

## 📁 技术预研

### WebGazer.js
- 仓库：https://github.com/brownhci/WebGazer
- 精度：校准后约 100-200px
- 输入：WebCam 视频流
- 输出：屏幕坐标 (x, y) 实时流
- 许可证：BSD

### MediaPipe Face Mesh
- 仓库：https://github.com/google-ai-edge/mediapipe
- 输出：面部468个3D关键点（含眼部轮廓、瞳孔）
- 可用于提升 WebGazer 精度（眼睛区域裁剪）

### Tobii Pro SDK
- 官网：https://www.tobii.com/
- Python SDK：`tobii_research`
- 精度：1-2px（完全能满足逐字级别的注视分析）
